What it does

One hub. Any brain. Your phone.

Attach one or several harnesses, watch them collaborate, and let them act on the real device — every sensitive action gated by consent on the phone.

Promote any harness to orchestrator

Give one harness the driver seat and it commands the rest — delegating subtasks to your other harnesses in parallel, even across several hubs, then merging it all into one answer.

Two live views

Watch every delegation as a file-system tree or a pannable flow graph — connectors light up green as results propagate back up the chain.

Real phone control

Camera, location, SMS, notifications, torch, ring, clipboard — and read & tap any app's screen via accessibility computer-use.

Take a photo, it sees it

The agent snaps a photo on your phone, it renders in the chat, and the model receives the actual pixels — so it can describe and reason about what it sees.

Talk or speak

Type from the phone, or use the always-on offline wake word and hold-to-talk; replies come back in chat or spoken aloud.

Consent & encryption

Per-harness × per-capability allow / ask / deny with biometric confirm. The relay only ever sees opaque ciphertext routed by fingerprint.

Phone superpowers

Just ask — it acts on your phone.

Your agent doesn't just chat. Tell it to use the real device — it takes the photo, reads the screen, sends the text — with every sensitive action gated by consent on the phone.

“Take a photo and tell me what's on my desk” “What notifications do I have?” “Text Alex I'm running 10 minutes late” “Where am I right now?” “Ring my phone — I can't find it” “Open the Camera app and switch to night mode”
Structured APIs Tier 1

Camera, location, SMS, notifications (listen & post), device info, battery, flashlight, ring, vibrate, clipboard, launch apps, photos — fast, typed, reliable.

Computer-use Tier 2

Drive any app's UI through the accessibility service: ui.read, ui.tap, ui.text, ui.swipe, ui.screenshot, back / home / recents — when there's no API, it uses the screen like you would.

The phone app

Your phone is in the loop, not just the target.

A native Kotlin/Compose app: rich chat with markdown, code, inline images and tables; multiple hubs and sessions; per-harness themes; and a full consent engine. The agent reaches you here — and you reach it back.

  • Pair several hubs, stay connected to all at once
  • Offline wake word, hold-to-talk, spoken replies
  • Auto-reconnect, boot-start, push wake doorbell
Security & access

Yours, end to end — reachable from anywhere.

It's your phone and your data. Nothing runs through someone else's cloud, every action needs your say-so, and you can still reach it from across the world.

End-to-end encrypted

Messages, photos and screen data are sealed on-device. The relay only ever sees opaque ciphertext addressed by fingerprint — it can't read your traffic, and neither can anyone who runs it.

Consent on every action

A per-harness × per-capability engine — allow, ask, or deny — with biometric confirm for sensitive actions. A harness can only do what you've permitted, capability by capability.

No accounts, no API key

Self-hosted on your own machine. Auth lives in your own CLI (your Claude subscription) — there's no API key in the project and no sign-up to use it.

Reach it from anywhere with Tailscale

Pair once, then drive your phone from any network — coffee shop, office, another country. Tailscale puts the hub on your private tailnet, so the connection works everywhere while the port stays off the public internet.

Get started

Running in a few minutes.

Needs Node 22+ and pnpm on your Mac/Linux, and an Android 10+ phone. Tailscale recommended so the phone reaches the hub from anywhere.

1 Clone & install

Bring up the hub

# clone and install
git clone https://github.com/rg1989/Agentic-Android
cd Agentic-Android/backbone && pnpm install && cd ..

# relay + hub (pick the agent in the UI)
make up
2 Open the hub

Pair & attach

# web UI: pair the phone, add a harness
make open   → http://127.0.0.1:8123

# or attach your own Claude (no API key)
make claude
3 Build the app

Install on the phone

# with the phone on USB + adb
make install

# then scan the pairing QR in the app
4 Orchestrate

Let them collaborate

# in Chat, set a harness to Orchestrator,
# then just ask — e.g.
"Delegate the security and perf review
 to two harnesses in parallel, each
 spawning its own sub-agents."
